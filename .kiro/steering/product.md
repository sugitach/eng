# Product Overview

## Product Overview
Emacsユーザーを対象とした次世代エディタ。Emacsの強力な機能と操作性を維持しつつ、現代的な開発環境に適合するパフォーマンス、拡張性、およびユーザーエクスペリエンスを提供することを目的とする。Rustによる高いパフォーマンスと分散アーキテクチャによる柔軟なUI対応を実現し、AI Agentとの連携もサポートする。

## Core Features
- Emacs互換の操作性（キーバインド、カーソル移動、リージョン操作など）
- 高いカスタマイズ性と拡張性（拡張言語による機能追加）
- 現代的なパフォーマンスと安定性（Zed準拠の速度目標）
- 統合開発環境 (IDE) 機能（コード補完、デバッグ、バージョン管理統合）
- Emacsの主要な利便性の維持と現代的な操作性の融合（バッファ/ウィンドウ/ペイン概念、killring、GUI OSのClipboard連携）
- 分散型アーキテクチャと多様なUIサポート（ターミナル、ネイティブGUI、Webブラウザ）
- dired, ediff, magitと同等の機能統合
- 入力ソース管理（FEP, IMなど）
- 国際化対応 (i18n)
- UTF-8ネイティブサポートとバイナリファイル編集モード
- AI Agentとの連携（コード生成、レビュー、質問応答など）
- Emacs Lispから拡張言語へのトランスレータ（優先度低）

## Target Use Case
- 既存のEmacsユーザーが、現代的な開発環境の要求に応える高性能なエディタに移行したい場合。
- 大規模なコードベースや複雑なプロジェクトを扱う開発者。
- リモート開発環境や多様なデバイス（デスクトップ、Web、ターミナル）で一貫したエディタ体験を求める開発者。
- AIを活用した開発支援を求める開発者。

## Key Value Proposition
- **Emacsの操作性を継承**: 長年のEmacs利用で培った知識と習慣を無駄にしない。
- **現代的なパフォーマンス**: RustとZedの技術を活用した高速かつ安定した動作。
- **柔軟な拡張性**: Elispに限定されない多様な拡張言語サポートと深いカスタマイズ性。
- **分散型アーキテクチャ**: 任意のUIとバックエンドの組み合わせによる柔軟な利用形態。
- **AIによる開発支援**: AI Agentとの連携による生産性向上。

## Terminology
本プロジェクトでは、以下の用語をEmacsの概念に基づいて定義し使用する。OSやGUIフレームワークの用語と区別すること。

- **Buffer (バッファ)**: 編集対象のテキストデータとその属性（ファイルパス、変更フラグ等）を保持するオブジェクト。`eng-core` プロセスで管理される。
- **View (ビュー)**: バッファを画面上に表示する矩形領域。Emacsにおける「Window」に相当する。カーソル位置やスクロール位置などの表示状態を持つ。
- **Window (ウィンドウ)**: OSのデスクトップ上に表示されるアプリケーションウィンドウ枠。Emacsにおける「Frame」に相当する。内部に1つ以上のViewを配置する。
